## 约束功能重构说明

### 🔍 原有问题

#### 问题 1：总预算约束未生效
```python
# UI 有输入
total_budget = st.number_input(...) if use_budget else None

# 但 optimization.py 根本没用这个参数！
def optimize_allocation_v2(model, base_x, y_target, total_budget=None, ...):
    # total_budget 从未被使用
```

#### 问题 2：约束设置体验差
- **6个渠道 = 12个输入框**（每个渠道2个）
- 没有批量设置功能
- 默认值不直观（基准×0.5 和 ×2.0）
- 无法快速设置常见场景

#### 问题 3：逻辑混乱
- `X_train_max` 既是默认值又是硬约束
- 用户可能设置超过训练范围的值（不合理）
- 没有"只增不减"、"固定某渠道"等常见需求的快捷方式

#### 问题 4：约束不符合业务场景
```
实际需求                     原有实现
- "只增不减"          →      需要12个输入框手动设置
- "某渠道固定"        →      无法快速设置
- "保守调整±30%"     →      需要手动计算每个值
- "尊重历史最大值"    →      逻辑混在代码里
```

### ✨ 重构后的改进

#### 改进 1：三种约束模式

| 模式 | 适用场景 | 特点 |
|------|---------|------|
| **无约束** | 探索最大潜力 | 完全自由优化 |
| **智能约束** ⭐推荐 | 日常使用 | 自动设置合理范围 |
| **自定义约束** | 特殊需求 | 完全手动控制 |

#### 改进 2：智能约束模式（新增）

**3个简单选项搞定所有设置：**

```python
1. 调整幅度：保守±30% / 适中±50% / 激进±100%
2. 尊重历史范围：✓ 不超过训练数据最大值
3. 允许归零：✓ 允许某些渠道降为0
```

**自动计算逻辑：**
```python
# 下限
if allow_zero:
    min_val = 0
else:
    min_val = base_x * (1 - range_factor)

# 上限
max_val = base_x * (1 + range_factor)
if respect_history:
    max_val = min(max_val, X_train_max)
```

#### 改进 3：批量设置（自定义模式）

**4个快捷按钮：**
```
┌─────────────────────┬─────────────────────┐
│ 全部设为基准±50%      │ 固定所有渠道         │
│ 仅允许增加（不减少）   │ 重置为训练范围       │
└─────────────────────┴─────────────────────┘
```

一键应用常见场景，无需手动输入12个值！

### 🔧 技术实现

#### 智能约束计算
```python
# 根据用户选择动态计算
range_factor = {"保守±30%": 0.3, "适中±50%": 0.5, "激进±100%": 1.0}[adjust_range]

for i in range(len(features)):
    min_val = 0 if allow_zero else base_x[i] * (1 - range_factor)
    max_val = base_x[i] * (1 + range_factor)
    
    if respect_history:
        max_val = min(max_val, X_train_max[i])
    
    min_constraints.append(min_val)
    max_constraints.append(max_val)
```

#### 批量设置实现
```python
if st.button("仅允许增加"):
    for i in range(len(features)):
        st.session_state[f"custom_min_{i}"] = base_x[i]
        st.session_state[f"custom_max_{i}"] = X_train_max[i]
    st.rerun()
```
