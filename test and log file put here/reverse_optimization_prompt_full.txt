【反推优化器改造方案：Claude 实施 Prompt】

以下是完整、可直接交给 Claude（或 VSCode Agent）的实施 Prompt。

------------------------------------------------------------------------

📌 任务背景

我有一个广告预算反推模块，它依赖一个黑盒模型 f(x) 来预测
KPI。当前实现基于：

-   中心差分估计各渠道局部敏感度（J_i）
-   一阶泰勒展开 Δy ≈ J·Δx
-   线性/约束求解 Δx
-   必要时 trust-constr 兜底

我现在需要对该模块进行升级，使其更稳定、更符合全局贡献。

------------------------------------------------------------------------

📌 目标：实现 “中心差分方向 + Shapley 校准幅度” 的反推框架

需要严格实现：

-   方向（sign）来自中心差分
-   幅度（magnitude）来自 Shapley 边际贡献 Δφ_i

即最终的敏感度向量为：

    gradient_i = sign(central_diff_i) * abs(shap_delta_i)

------------------------------------------------------------------------

📌 新旧方案差异

  版本           方向来源   幅度来源
  -------------- ---------- --------------
  旧版           中心差分   中心差分
  新版（目标）   中心差分   Shap Δφ 校准

理由：

-   中心差分方向 = 当前局部真实斜率 → 决定该往哪个方向调预算
-   Shap Δφ = 全局平均贡献 → 决定该调多少更稳

因此必须保留差分方向，但让幅度来自 Shapley。

------------------------------------------------------------------------

📌 Claude 需要按以下步骤修改代码

Step 1. 继续使用中心差分计算 J_i（不变）

只取方向：

    direction_i = sign(J_i)

不再用 J_i 本身作为幅度。

------------------------------------------------------------------------

Step 2. 新增 Shapley 计算 Δφ_i

需要：

-   在当前样本 x 上运行 TreeExplainer 或 KernelExplainer
-   获取每个渠道的 Shap φ_i
-   作为 Δφ_i（即边际贡献）

新增：

    def compute_shap_marginal(model, x): 
        return shap_values

------------------------------------------------------------------------

Step 3. 构建校准后的敏感度（幅度）

使用：

    sensitivity_i = abs(shap_delta_i)

或混合校准（可选）：

    sensitivity_i = abs(J_i) * w1 + abs(shap_delta_i) * w2

默认 w1=0, w2=1。

------------------------------------------------------------------------

Step 4. 生成最终的梯度向量

    gradient_i = direction_i * sensitivity_i

之后优化器仍然使用 Δy ≈ gradient · Δx。

------------------------------------------------------------------------

Step 5. 替换所有优化器内部对 J_i 的依赖

所有之前写成：

    gradient = J

全部替换成：

    gradient = direction * sensitivity

不改其余约束/线搜索流程。

------------------------------------------------------------------------

Step 6. 加入处理边界与性能的逻辑

包括：

1.  Shap 计算慢 → 仅对 top-k 渠道计算 Shap
2.  当方向冲突（sign(J_i) 与 sign(φ_i) 不一致）时：
    -   方向永远以中心差分为准
3.  预算大步长调整时 → 用线搜索控制步幅
