## 📖 项目概述

**Ad Effect Intelligence** 是一个专为广告投放效果分析设计的智能决策支持系统。通过机器学习模型和 SHAP（SHapley Additive exPlanations）可解释性分析，帮助广告主：

- 🎯 **理解投放效果**：识别哪些渠道或因素对广告效果影响最大
- 📊 **优化预算分配**：基于数据驱动的方法，智能推荐预算分配方案
- 🔍 **反向推理**：设定目标效果，反向计算所需的投放策略
- 💡 **可视化洞察**：直观展示特征重要性、影响方向和贡献占比

### 适用场景

- 多渠道广告投放效果分析
- 营销预算优化与分配
- ROI 预测与目标倒推
- 广告策略评估与调整

## ⚡ 核心功能

### 1️⃣ 数据上传与预览

- 支持 **CSV** 和 **Excel** 格式数据导入
- 自动识别数值型特征
- 数据基本统计信息展示
- 缺失值和异常值检测

### 2️⃣ 模型训练与评估

- **随机森林模型**训练（可扩展至 XGBoost、LightGBM）
- 灵活的特征选择和目标变量设置
- 全面的模型评估指标：
  - R²（决定系数）
  - RMSE / NRMSE（归一化均方根误差）
  - 交叉验证评分
- **智能评分系统**：综合评估模型质量并给出使用建议

### 3️⃣ 可视化分析

基于 **SHAP** 的可解释性分析，包括：

- **Summary Plot**：特征重要性排序和影响方向分布
- **Waterfall Plot**：单样本预测贡献分解
- **贡献占比饼图**：直观展示各特征对整体效果的贡献
- **特征相关性热力图**：识别多重共线性问题

### 4️⃣ 反推与预算优化

核心创新功能，支持三种优化模式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **线性分配法** | 快速估算，无约束 | 初步评估、趋势预测 |
| **预算约束优化** | 固定总预算，优化分配 | 预算固定的投放场景 |
| **全约束优化** | 加入平滑项，控制波动 | 实际执行阶段，稳定性优先 |

**核心算法特点**：
- 直接求解法，计算速度快
- 基于 SHAP 权重和敏感度的智能分配
- 目标可达性分析和预警机制
- 详细的优化结果对比和建议

## 📚 详细使用指南

### 数据准备要求

#### 数据格式
- 支持 `.csv` 或 `.xlsx` / `.xls` 格式
- 需包含表头（列名）
- 至少包含 **2 个数值型特征列** 和 **1 个目标列**

#### 数据示例

| 渠道A投放 | 渠道B投放 | 渠道C投放 | 广告效果 |
|----------|----------|----------|---------|
| 1000     | 1500     | 800      | 3500    |
| 1200     | 1300     | 900      | 3800    |
| 900      | 1600     | 1000     | 3900    |

> **提示**：特征列通常是可控变量（如投放预算、曝光量），目标列是结果变量（如转化量、销售额）

### 模型训练

1. **选择目标变量**：从下拉列表中选择你要预测的列
2. **选择特征变量**：勾选用于训练的输入特征
3. **设置随机种子**（可选）：保证结果可复现，默认 42
4. **点击训练**：等待模型训练完成

### 模型评估指标说明

| 指标 | 含义 | 评价标准 |
|------|------|----------|
| **训练集 R²** | 模型在训练数据上的拟合程度 | 越接近 1 越好 |
| **测试集 R²** | 模型在新数据上的泛化能力 | **最重要**，越接近 1 越好 |
| **交叉验证 R²** | 模型稳定性评估 | 与测试集 R² 接近为佳 |
| **NRMSE** | 归一化误差 | 越小越好（< 0.1 为优秀） |

**模型评分系统**：
- **90+ 分**：优秀，可直接用于决策
- **80-90 分**：很好，推荐使用
- **70-80 分**：良好，建议验证后使用
- **< 70 分**：需改进

### SHAP 可视化分析

#### Summary Plot（特征重要性图）
- **纵轴**：特征名称，按重要性排序
- **横轴**：SHAP 值（对预测的影响）
- **颜色**：特征值大小（红=高，蓝=低）

**解读方法**：
- SHAP 值 > 0 → 该特征推高了预测结果
- SHAP 值 < 0 → 该特征降低了预测结果
- 点的分布越分散 → 特征影响越不确定

#### 贡献占比饼图
直观展示各特征对模型预测的平均贡献百分比

### 预算优化使用方法

#### 基本使用

1. **设定目标值**：输入期望达到的广告效果
2. **选择优化模式**：
   - 无约束（快速估算）
   - 预算约束（总预算固定）
   - 全约束（包含上下限）
3. **运行优化**：点击"开始优化"

#### 结果解读

优化结果表包含以下字段：

| 字段 | 说明 |
|------|------|
| **基准投放** | 当前平均投放水平 |
| **建议投放** | 优化后的推荐值 |
| **增量** | 需要调整的幅度 |
| **敏感度** | 该渠道对效果的影响程度 |
| **SHAP 权重** | 特征重要性占比 |

#### 使用建议

> **⚠️ 适用性提醒**
> 
> 反向推理功能在以下条件下效果最佳：
> - 特征是**可控的**（如广告预算、投放时长）
> - 因果关系是**直接的**（特征直接影响目标）
> - 优化结果是**可执行的**（不存在外部限制）

## 🔧 开发说明

### 自定义配置

编辑 `config.py` 文件可修改：

```python
# 修改应用密码
APP_PASSWORD = "your_password"

# 修改页面标题
PAGE_CONFIG = {
    "page_title": "Your Title",
    "page_icon": "🎯",
    ...
}

# 自定义 CSS 样式
CSS_STYLE = """
<style>
    /* 你的样式 */
</style>
"""
```

### 添加新页面

1. 在 `pages_module/` 下创建新的 Python 文件
2. 定义页面函数：

```python
import streamlit as st

def page_your_feature(state):
    st.header("新功能标题")
    # 你的页面逻辑
```

3. 在 `shap_streamlit_app.py` 中导入并注册：

```python
from pages_module.your_module import page_your_feature

# 添加到导航
page = st.radio("导航", (..., "新功能名称"))

# 添加路由
elif page == "新功能名称":
    page_your_feature(state)
```

### 扩展模型支持

当前支持随机森林，可通过修改 `pages_module/model_training.py` 添加其他模型：

```python
from lightgbm import LGBMRegressor
from xgboost import XGBRegressor

# 添加模型选择
model_type = st.selectbox("选择模型", ["随机森林", "LightGBM", "XGBoost"])

if model_type == "LightGBM":
    model = LGBMRegressor(...)
elif model_type == "XGBoost":
    model = XGBRegressor(...)
```

## ❓ 常见问题

### Q1: 上传数据后提示"没有足够的数值列"怎么办？

**A**: 确保数据集中至少有 3 列数值型数据（2 个特征 + 1 个目标）。文本列会被自动忽略。

### Q2: 模型评分很低（< 60 分）是什么原因？

**A**: 可能的原因：
1. 特征与目标变量相关性弱
2. 数据量太少（建议 > 50 行）
3. 数据质量问题（缺失值、异常值）
4. 需要特征工程（如添加交互项、多项式特征）

**建议**：检查交叉验证 R² 和测试集 R²，如果都很低，说明数据本身可预测性不强。

### Q3: 优化结果显示"目标值过高"怎么办？

**A**: 这表示即使所有特征都调整到最大值，模型预测也无法达到目标。你可以：
1. 降低目标值
2. 检查特征上限设置是否过于保守
3. 考虑添加更多特征

### Q4: 为什么优化后的预测值与目标值不完全一致？

**A**: 这是正常的，原因：
1. 模型是非线性的，线性近似存在误差
2. 存在约束限制（预算、上下限）
3. 优化算法寻找的是可行解而非精确解

如果偏差 < 5%，通常是可接受的。

### Q5: 如何导出优化结果？

**A**: 可以使用 Streamlit 的内置功能：
1. 选中表格数据并复制
2. 或者使用截图工具保存可视化图表
3. 未来版本会添加一键导出功能

### Q6: 能否在生产环境部署？

**A**: 可以，建议：
1. 使用 Docker 容器化部署
2. 配置反向代理（Nginx）
3. 启用 HTTPS
4. 定期备份模型和数据
5. 监控应用性能
